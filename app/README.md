## Листинг файла `activity_main.xml` с изменениями

**Верхняя надпись:** 

1. В файле  `strings.xml` создал строковый ресурс `greeting` с соответствующим текстом и присвоил его с помощью атрибута `text`.
2. Установил синий цвет через атрибут `textColor`.
3. Увеличил размер до 24sp с помощью атрибута `textSize`.
 ```xml
   <TextView
        android:text="@string/greeting"
        android:textColor="#0000FF"
        android:textSize="24sp"
       ... />
   ```

**Нижнаяя надпись:** 

1. Создал строковую переменную `additional_greeting` с другим текстом.
2. Поменял цвет на красный.
3. Указал размер 21sp.
4. Добавил отступ 70 dp сверху через атрибут `layout_marginTop`:
```xml
   <TextView
        android:text="@string/additional_greeting"
        android:textColor="#FF0000"
        android:textSize="21sp"
        android:layout_marginTop="70dp"
       ... />
   ```

## Ответы на контрольные вопросы

**1. Какие основные компоненты входят в структуру Android-проекта?**

- `manifests/` - содержит файл `AndroidManifest.xml`.
- `kotlin + java/` - содержит исходный код приложения `MainActivity`.
- `res/` - содержит ресурсы приложения: 
    - изображения `drawable/`.
    - макеты `layout/` (`activity_main.xml`).
    - иконки `mipmap/`.
    - значения `values/`.
        - темы `themes/`.
        - цвета `colors.xml`.
        - строки `strings.xml`.
- `Gradle Scripts` - содержит файлы сборки проекта (`build.gradle.kts`).
<br><br>

**2. Для чего нужен файл `AndroidManifest.xml`?**

Данный файл содержит основную информацию о приложении, его компонентах и настройках, необходимых для взаимодействия с Android. Является основным конфигурационным файлом приложения, в котором содержится имя пакета, список активностей и других компонентов, а также запрашиваемые разрешения.
<br><br>

**3. Чем отличается `minSdkVersion` от `targetSdkVersion`?**

- `minSdkVersion` - минимальная версия Android, на которой приложение может работать.
- `targetSdkVersion` - версия, на которой приложение тестировалось и для работы на которой оно оптимизировано.
<br><br>

**4. Что такое AVD и для чего он используется?**

Android Virtual Device - это эмулятор конкретного устройства Android (телефона, планшета, телевизора, умныых часов), который можно использовать для запуска и тестирования приложения без реального физического устройства.
<br><br>

**5. Как изменить текст приложения без изменения кода активности?**

С помощью файла ресурсов `res/values/strings.xml` необходимо изменить значение строковой переменной (либо создать новую), после этого текст обновится во всех местах, где используется ссылка на этот ресурс (например, `android:text="@string/имя_ресурса"`).
